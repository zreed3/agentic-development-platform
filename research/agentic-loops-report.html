What an agent loop is

Strip away the frameworks and an agent is just machinery wrapped around a stateless function. The loop feeds the model's own outputs — and the world's responses — back into its next input. The loop is the agent. Remove it and you have a single completion.

gather context→ take action→ verify work↻
the four forced questions
  • Context — what goes in the window this turn? (finite)
  • Action — what can it do, and how is that expressed?
  • Observation — how does the result re-enter the loop?
  • Termination — when, and on whose authority, does it stop?
the 2026 shift
  • harnesses > algorithms — the field hardens loops, it doesn't invent new ones
  • context engineering became the umbrella discipline
  • durable state lives in the filesystem & git, not the window
  • single-thread vs multi-agent is still unsettled
  • evaluation is now the bottleneck, not capability

Three vocabularies, one cycle: classical perceive→decide→act, ReAct's thought→action→observation, and Anthropic's gather→act→verify→repeat. The shape isn't new — what's new is that one pretrained model fills the "reason" slot and the action space is open-ended.

The live debate

Single-threaded linear agents, or orchestrator-worker swarms? In 2026 this is genuinely unsettled — the two strongest positions landed within ~24 hours of each other.

Cognition verified

"Don't build multi-agents"

  • parallel subagents can't share full context
  • actions carry implicit decisions → conflicting decisions → bad results
  • share full agent traces, not just messages
  • prefer single-threaded linear agents
vs

Anthropic verified

orchestrator-worker research system

  • lead agent spawns 3–5 parallel subagents
  • lead synthesises & decides if more work is needed
  • plan persisted to a Memory store past 200k tokens
  • +90.2% over single-agent Opus 4 — self-reported, internal, unreproduced
ADG takes no side

The subagent budget gate bounds the cost / blast-radius of fan-out (maxConcurrent, maxTotal) and leaves the topology choice to the operator — defaulting to warn so it advises before it blocks. It keeps the lab-specific 90.2% magnitude out of policy entirely.
