Quick Start

1. Start With SQL

Choose a feature and bounded packet before reading broadly.

pnpm context:lite -- --feature S30

2. Pick One Function

Open the matching skill in development/_system/skills. Use one skill per job so the context stays small.

3. Prove The Change

Run the narrowest relevant check, record failed commands, and add release evidence only when the claim is proved.

The 5/5 Rule

An area is 5/5 when:

  • The source of truth is SQL-queryable or machine-readable.
  • The human docs are generated from or anchored to that source.
  • Live surfaces include allowed, read-only, empty, error, forbidden, locked, and destructive-action states where relevant.
  • Protected operations prove entitlement, permission, tenant scope, business scope, audit behavior, and negative tests.

An agent is done only when:

  • The original requirement is mapped to source files and evidence.
  • The matching skill was used for the function being changed.
  • Checks cover the actual scope, not a smaller nearby claim.
  • Traceability evidence is appended for material process, route, UX, architecture, or release changes.

How To Use The Skills

Normal Use

Open exactly one skill folder for the function you are performing. The skill body should be enough to act without loading this whole wiki.

sed -n '1,180p' development/_system/skills/dev-rbac-entitlement-auditor/SKILL.md

Discoverable Use

These are complete Codex-style skill packages. To make one auto-discoverable for future Codex runs, mirror the folder into the repo or user skill location used by that environment.

Workflow

Context

Run the context skill and feature packet. Stop if the feature, route, or evidence path is unclear.

Contract

Write or update the smallest contract that proves the user experience, authority boundary, and test seam.

Build

Edit only scoped implementation files and directly related tests. Do not hand-edit generated mirrors.

Verify

Run targeted checks first. Use DB, security, browser, or traceability gates only when the touched boundary requires them.

Evidence

Record one consolidated event when the same proof covers multiple rows. Record failed checks as failures.

Drift Check

Regenerate planning docs and run traceability gates for scope, route, UX, architecture, integration, or release changes.
