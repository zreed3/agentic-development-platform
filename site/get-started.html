Installation & setup

Node, SQLite, and five minutes.

ADG has no runtime dependencies to install. Everything runs on Node ≥20 and the sqlite3 CLI. Clone the repo, run setup, and the full gate is green in seconds.

Dependencies 0 No npm install needed for the core platform. Node ≥20 and the sqlite3 CLI are the only requirements.
Overview / Get Started

Prerequisites

Requirements

Node ≥20

Any current LTS release will do. Check with node --version. The scripts use Node's built-in module system — no transpilation, no bundler.

sqlite3 CLI on PATH

The scripts shell out to sqlite3 — no Node database driver needed. Check with sqlite3 --version. Install via your OS package manager if missing.

# macOS
brew install sqlite

# Debian/Ubuntu
apt-get install sqlite3

Quickstart — this repo

Try it now

The fastest way to see ADG working is to run it on its own self-referential demo backlog.

# 1. Clone the repo
git clone https://github.com/otterblock/agentic-development-platform.git
cd agentic-development-platform

# 2. Build the demo database (self-referential ADG worked example)
npm run setup:demo

# 3. Run the full governance gate
npm run ci:governance

# 4. Generate a bounded context packet to see what agents receive
npm run context:feature -- --feature S07 --workflow route
npm run context:feature -- --feature S07 --workflow route --format toon

npm run setup vs npm run setup:demo: setup builds an intentionally empty database for a clean install. setup:demo loads the self-referential ADG worked example and mirrors the audit log into SQL — use this to explore the full system.

Adopting ADG in another repo

Host repo setup

The layer is deliberately small and side-effect-light. Removing it means deleting four directories and the related package.json scripts — nothing patches the host application at runtime.

  1. 1
    Copy the governance layer

    Copy config/, scripts/, tooling/, data/schema.sql, and AGENTS.md into the host repo.

  2. 2
    Replace the seed

    Edit data/seed/backlog.seed.json with the host project's epics, features, items, routes, personas, and integrations.

  3. 3
    Fill in the Project Profile

    Open AGENTS.md and replace the Project Profile section with product, surfaces/packages, RBAC model, integrations, and host-specific gates.

  4. 4
    Install the skills

    See the skills section below. Two portable skills encode the platform's traceability and build-runner disciplines.

  5. 5
    Run setup and validate

    npm run setup && npm run ci:governance. Add host-specific gates (route registry, typecheck, lint, tests) to the ci:governance script.

Keep *.sqlite on .gitignore. Generated databases are rebuilt by npm run setup. Committing them re-introduces the exact anti-pattern ADG was built to prevent, and adg:doctor will flag it.

Using the installer

adg:install

The installer copies the portable lane policy, classifier/guard, and docs into a host repo, adds the relevant package.json scripts, and records an install-state file so updates can detect what was installed.

# Install Proofline into a host repo
npm run adg:install -- --target /path/to/host-repo

# Install with Claude Code client support (plugin + CLAUDE.md + hooks)
npm run adg:install -- --target /path/to/host-repo --client claude

# Refresh managed files (backs up existing before replacing)
npm run adg:update -- --target /path/to/host-repo

# Check what is installed and whether it has drifted
npm run adg:install:status -- --target /path/to/host-repo --format json

--client claude installs/refreshes the Claude Code plugin, the generated CLAUDE.md, hooks, and a .claude/settings.json permissions template into the host repo. Existing unmanaged files are not overwritten by install unless --force is used.

Installing the skills

Codex & Claude Code

Two portable skills encode the platform's disciplines so they survive across repos and across agents.

agentic-traceability

Keeps the append-only audit log and SQL backlog current while building. Covers when to record an event, what evidence to attach, and how to structure decision events for waived gates.

agentic-build-runner

Executes backlog items end to end: select → claim → implement → test → record evidence → commit-ready. Loads agentic-traceability as a companion.

Install for Codex

Codex discovers skills under ~/.codex/skills. Symlink them so edits stay live:

ln -s "$(pwd)/skills/agentic-traceability" ~/.codex/skills/agentic-traceability
ln -s "$(pwd)/skills/agentic-build-runner"  ~/.codex/skills/agentic-build-runner

Install for Claude Code (via plugin marketplace)

# Add the ADG marketplace
claude plugin marketplace add https://github.com/otterblock/agentic-development-platform.git

# Install the governance plugin (includes skills + hooks + commands)
claude plugin install adg-governance

Use with other runtimes

The skills are plain Markdown with YAML frontmatter. Any runtime that loads Markdown instructions can use them — point your skill/instruction loader at the skills/ directory, or paste the relevant SKILL.md into the system prompt. The content is runtime-agnostic; only the discovery path differs.

# Invoke in a Codex session
$agentic-traceability
$agentic-build-runner
