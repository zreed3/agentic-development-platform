SQLite views

Derived state the plugin can trust

These views are declared by scripts/backlog-db.mjs and mirrored into data/schema.sql.

View What it shows Primary use
backlog_summary Feature rows with epic, priority, release band, task count, and test count. Backlog overview and next-work selection.
feature_current_status Feature planned status plus latest status and summary from audit events. Truthful feature state without rewriting feature rows.
backlog_item_current_status Item lifecycle status, latest event, active claim actor, expiry, and write scope. Claim/start/complete/verify coordination.
feature_tasks Task-only slice from feature_items. Implementation work packets.
feature_test_cases Test-case slice from feature_items. Targeted verification planning.
feature_success_criteria Success-criterion slice from feature_items. Completion evidence checks.
feature_use_cases Use-case slice from feature_items. Requirement and UX lineage.
persona_workflows Persona, RBAC role, realm, access level, route, and expected state per feature. Route/persona truth passes and UX-as-code checks.

SQL workbench

Copyable local queries

The HTML file stays static. Copy a command and run it from the repository root after npm run setup or npm run setup:demo.

Feature status board

sqlite3 
sqlite3 data/backlog.sqlite "select feature_id,title,planned_status,current_status,latest_update_at,latest_update from feature_current_status order by feature_id;"

Backlog summary

sqlite3 data/backlog.sqlite "select id,epic,title,status,release_band,task_count,test_count from backlog_summary order by id;"

Item lifecycle board

sqlite3 data/backlog.sqlite "select id,feature_id,item_type,current_status,active_claim_actor,active_claim_expires_at from backlog_item_current_status order by feature_id,position;"

Plugin command map

Read-only controls exposed to Codex

The neutral manifest keeps the plugin portable: each command names its script, output contract, and guardrail risk class.

Command Script Risk Output shape
elicitation-graph node scripts/adg-elicitation.mjs graph --feature <feature> --format json read-only Requirements graph nodes, edges, and dangling-edge failures.
context-slice node scripts/adg-context.mjs slice --feature <feature> --workflow <workflow> read-only Bounded packet, efficiency metrics, and stop conditions.
ux-validate node scripts/adg-ux.mjs validate --feature <feature> read-only Coverage, journey specificity, evidence, and RBAC/persona gaps.
ux-truth-pass node scripts/adg-ux.mjs truth-pass --feature <feature> --format markdown --check read-only Lifecycle classification, route/persona matrix, and downgrade recommendations.
ux-downgrade-report node scripts/adg-ux.mjs downgrade --feature <feature> --format json read-only Audit-ready downgrade recommendations for failing route/persona expectations.
standards-matrix node scripts/adg-standards.mjs matrix --format json read-only Standards controls, evidence links, and maturity domains.
deliverable-audit node scripts/adg-deliverable.mjs audit read-only Deliverable evidence records and traceability failures.
