Agentic Development Governance (ADG)

A GitHub-ready setup guide for running agent-assisted development with local guardrails, bounded context, an append-only audit trail, and SQL-first backlog governance.

Requirements

The platform is deliberately small. You need a shell, Node.js 20 or newer, and the SQLite command-line tool. There are no runtime dependencies to install.

Node

Node.js 20+

All governance scripts run with the built-in Node runtime.

SQLite

sqlite3 on PATH

The backlog database is generated locally and queried from the CLI.

Manual install

ADG is not distributed as an npm package right now. Install it by copying or cloning the repository into your development folder, then run the local npm scripts that ship with the repo.

1

Copy it into development

Use GitHub clone, a downloaded zip, or a local folder copy.

mkdir -p ~/Development
cd ~/Development
git clone <repo-url> agentic-development-governance
cd agentic-development-governance
2

Build the database

Regenerate data/backlog.sqlite from source files.

npm run setup
3

Run the full gate

Validate backlog, audit, guardrails, evals, metrics, and broker.

npm run ci:governance
4

Ask for bounded context

Use SQL-selected packets before opening source files.

npm run context:feature -- --feature S07 --workflow route
npm run context:item -- --item S07-TASK-01 --workflow route --format toon

Install the agent skills

The portable skills teach an agent to keep the SQL backlog and append-only audit trail current while it works. Symlink them for live edits, or copy them for a frozen snapshot.

Symlink for live edits

mkdir -p ~/.codex/skills
ln -s "$(pwd)/skills/agentic-traceability" ~/.codex/skills/agentic-traceability
ln -s "$(pwd)/skills/agentic-build-runner" ~/.codex/skills/agentic-build-runner

Copy for a snapshot

mkdir -p ~/.codex/skills
cp -R skills/agentic-traceability ~/.codex/skills/
cp -R skills/agentic-build-runner ~/.codex/skills/

Use the skills during work

Invoke $agentic-traceability when recording audit or backlog evidence. Invoke $agentic-build-runner when an agent should claim, build, test, complete, and verify backlog work.

Adopt in another repo

Copy the governance layer into a host project, replace the demo backlog with the host project's real plan, and fill in the Project Profile in AGENTS.md. 

cp -R config scripts tooling data/schema.sql AGENTS.md <host-repo>/
cp -R skills <host-repo>/

cd <host-repo>
# Replace data/seed/backlog.seed.json with the host project's features and routes.
# Fill in the Project Profile section in AGENTS.md.
npm run setup
npm run ci:governance
